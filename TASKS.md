# 작업 대시보드

> 이 파일은 할 일, 결정 사항, 미해결 질문을 한곳에서 관리하는 초안입니다.  
> 항목을 추가할 때는 `- [ ] **제목** - 맥락, 기한` 형식을 유지하세요.

## 오늘의 초점

- [ ] **이번 주 가장 중요한 결과 정하기** - 예: 유료 고객 인터뷰 3건 완료
- [ ] **다음 행동 1개 실행하기** - 예: 잠재 고객에게 인터뷰 요청 메시지 보내기

## 진행 중

- [ ] **AI 비즈니스 아이디어 검증하기** - 타깃 고객, 문제, 지불 의사를 짧게 정리
  - 다음 검토: 2026-07-22
- [ ] **업무 대시보드 습관 만들기** - 매일 시작 전 5분, 마감 전 5분 업데이트

## 대기 중

- [ ] **고객 피드백 회신 대기** - 인터뷰 요청 발송 후 기록
  - 대기 시작일: 2026-07-19

## 결정 사항

|---|---|---|---|
| 2026-07-19 | Slack 없이 파일 기반 대시보드로 시작 | 도구 연결 없이 즉시 사용 가능 | `TASKS.md`에 업무를 기록 |
| YYYY-MM-DD | 결정 내용 | 판단 근거 | 실행할 일 |

|---|---|---|---|
| 첫 고객은 누구인가? | 판매 메시지와 채널을 정하는 기준 | 후보 10명 목록 작성 후 인터뷰 | 2026-07-26 |
| 가장 먼저 검증할 문제는 무엇인가? | 제품 범위를 좁히기 위해 | 인터뷰에서 반복 언급되는 문제 확인 | 2026-07-26 |
| 질문 내용 | 중요 이유 | 확인 방법 | YYYY-MM-DD |

### 빠른 사용법

1. 새 할 일은 `오늘의 초점` 또는 `진행 중`에 추가합니다.
2. 막힌 일은 `대기 중`으로 옮기고, 누구 또는 무엇을 기다리는지 적습니다.
3. 중요한 판단은 `결정 사항`에 한 줄로 남깁니다.
4. 답이 필요한 내용은 `미해결 질문`에 기록하고 기한을 정합니다.
5. 끝난 일은 `완료`로 옮겨 `[x]`로 표시합니다.
